# Build Orders

Build orders should respond to the map. A memorized order is useful only when it matches your resource access, faction plan, and next fights.

## Stable priorities

1. Enable early fights with enough recruits.
2. Secure recurring growth.
3. Fix the resource bottleneck blocking the next dwelling.
4. Add upgrades when they change fight quality.
5. Add magic or economy when the payoff arrives before the decisive window.

## Timing checks

- What can I build today?
- What resource blocks tomorrow?
- Which mine or pile fixes that block?
- Does the next building produce weekly value?
- Does an upgrade prevent more losses than extra growth would?

## When to deviate

Deviate when the map gives early rare resources, a guarded bank is unusually cheap, an opponent is close, or a spell building unlocks a fight the army cannot otherwise take.
