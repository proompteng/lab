### Summary
Implement the Temporal Bun SDK payload DataConverter module, wire it through client and workflow runtimes, and update tests/docs so payload handling no longer relies on ad-hoc JSON.
### Steps
1. Create payload converter primitives under `packages/temporal-bun-sdk/src/common/payloads` (e.g. `converter.ts`, `json-codec.ts`, `failure.ts`, `index.ts`) implementing the DataConverter design from [packages/temporal-bun-sdk/docs/payloads-codec.md](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/docs/payloads-codec.md); export via `packages/temporal-bun-sdk/src/common/index.ts` and add new `bun:test` suites under `packages/temporal-bun-sdk/tests/payloads` covering JSON round-trips, metadata tunnelling, and failure helpers.
2. Thread the converter through the client by updating [packages/temporal-bun-sdk/src/client/serialization.ts](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/src/client/serialization.ts) and [packages/temporal-bun-sdk/src/client.ts](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/src/client.ts) to accept a `DataConverter`, encoding args/memo/headers/search attributes and query responses deterministically (including hashing in `computeSignalRequestId`); adjust [packages/temporal-bun-sdk/src/client/types.ts](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/src/client/types.ts) option types, refactor [packages/temporal-bun-sdk/src/internal/core-bridge/native.ts](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/src/internal/core-bridge/native.ts) to surface raw payload bytes instead of JSON, and extend `packages/temporal-bun-sdk/tests/client.test.ts` plus new targeted specs to cover default and custom codecs.
3. Inject the converter into runtime paths by replacing `defaultPayloadConverter`/`defaultFailureConverter` usage in [packages/temporal-bun-sdk/src/workflow/runtime/environment.ts](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/src/workflow/runtime/environment.ts) and [packages/temporal-bun-sdk/src/worker/runtime.ts](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/src/worker/runtime.ts); plumb configurability through worker/workflow factories and ensure activation, activity, and failure handling use the injected converter, with updated assertions in `packages/temporal-bun-sdk/tests/worker.runtime.workflow.test.ts` and related fixtures.
4. Refresh surface area and docs: update `packages/temporal-bun-sdk/src/index.ts`, [packages/temporal-bun-sdk/docs/client-runtime.md](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/docs/client-runtime.md), [packages/temporal-bun-sdk/docs/workflow-runtime.md](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/docs/workflow-runtime.md), [packages/temporal-bun-sdk/README.md](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/README.md), and [packages/temporal-bun-sdk/docs/testing-plan.md](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/docs/testing-plan.md) to describe configuring the converter and document the JSON-tunnel limitation; extend [packages/temporal-bun-sdk/tests/native.integration.test.ts](https://github.com/proompteng/lab/blob/main/packages/temporal-bun-sdk/tests/native.integration.test.ts) with a custom codec scenario exercising workflow start/signal/query/failure under `TEMPORAL_TEST_SERVER=1`.
### Validation
- From `/workspace/lab`, run `pnpm exec biome check packages/temporal-bun-sdk/src packages/temporal-bun-sdk/tests packages/temporal-bun-sdk/docs`.
- Run `pnpm --filter @proompteng/temporal-bun-sdk run build`.
- Run `pnpm --filter @proompteng/temporal-bun-sdk test`.
- With the Temporal CLI server running, run `TEMPORAL_TEST_SERVER=1 pnpm --filter @proompteng/temporal-bun-sdk exec bun test tests/native.integration.test.ts`.
### Risks
- Signal and query request IDs must remain deterministic after payload encoding; regressions could break idempotency or dedupe guarantees.
- The Zig bridge still expects JSON inputs, so the converter must tunnel codec metadata safely without native changes; misalignment here could corrupt payloads or fail replays.
- Integration coverage depends on a local Temporal dev server; provisioning or flaky startup will block validation and needs early coordination.
