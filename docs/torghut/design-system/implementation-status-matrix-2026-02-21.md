## Strict Status Rules

- `Implemented`: verified evidence for all three categories: implementation code, targeted tests, and runtime/config (or migration) artifact.
- `Partial`: implementation exists or overlaps, but strict evidence is incomplete for full doc scope.
- `Planned`: document is index/snapshot/roadmap level; design intent exists but feature set is not fully implemented in code/runtime.
- `Planned`: index/meta/snapshot document; not a direct implementation target.
